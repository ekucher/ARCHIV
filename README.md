# ARCHIV_VETOFFICE

Автоматизована система резервного копіювання для VetOffice з підтримкою архівації, перевірки цілісності, контролю вільного місця та завантаження резервних копій на SFTP-сервер.

---

# Версія

**ARCHIV_VETOFFICE v2.6**
Дата релізу: **2026-05-31**

---

# Основні можливості

## Архівація

* Створення архівів через 7-Zip.
* Підтримка паролів архівів.
* Підтримка SHA256 CRC.
* Багатопотокове стиснення.
* Перевірка архіву після створення (`7z t`).
* Контроль мінімального вільного місця перед архівацією.
* Автоматичне формування імен файлів.

Приклад:

```text
vetcontrol_dev_pnmgu_v2508_20260530_1657.mdz
```

---

## Контроль цілісності

Для кожного архіву автоматично створюється:

```text
archive.mdz
archive.mdz.sha512
```

Після створення виконується перевірка:

```text
Контрольна сума SHA512 збігається
```

## Розширена валідація архівів

Після створення архіву скрипт виконує додаткові перевірки перед копіюванням або завантаженням на SFTP.

### Перевірка розміру архіву

Скрипт порівнює розмір створеного архіву з розміром джерела та перевіряє мінімальні пороги:

```text
--- ПЕРЕВIРКА РОЗМIРУ АРХIВУ VETOFFICE ---
[INFO] Перевiрка розмiру архiву VETOFFICE: архiв=32.91 MB; джерело=505.96 MB; частка=6.505%
[SUCCESS] Перевiрка розмiру архiву пройдена: VETOFFICE
```

Основні параметри:

```powershell
$enableArchiveSizeValidation = $true
$minimumArchiveSizeMB = 1
$minimumArchivePercentOfSource = 0.1
```

Якщо архів підозріло малий, подальше копіювання або SFTP-завантаження для цієї категорії не виконується.

### Test Restore

Опційно можна увімкнути тестове відновлення архіву у тимчасову папку.

```powershell
$enableArchiveTestRestore = $false
$archiveTestRestoreTempPath = "$env:TEMP\ARCHIV_VETOFFICE_TEST_RESTORE"
```

У стандартному режимі `Test Restore` вимкнений, щоб не збільшувати час щоденного бекапу. Якщо параметр увімкнено, скрипт розпаковує архів у тимчасовий каталог, перевіряє наявність відновлених файлів і після перевірки очищає тимчасові дані.

### Загальний статус валідації

Для кожної категорії формується підсумковий статус:

```text
ArchiveValidationSuccess
```

Архів вважається готовим до копіювання або SFTP-завантаження тільки якщо успішно пройшли:

* створення архіву;
* створення SHA512;
* перевірка SHA512;
* перевірка розміру архіву;
* Test Restore, якщо він увімкнений.

## Особливості

### Автоматичне визначення StorageBox Host

Можна вказати:

```powershell
SftpHost = "u363066"
```

Скрипт автоматично перетворить на:

```text
u363066.your-storagebox.de
```

---

### Автоматичне створення каталогів

Перед завантаженням виконується:

```text
mkdir archiv
mkdir blog
```

Якщо каталог уже існує:

```text
File already exists
```

помилка ігнорується автоматично.

---

### Автоматичне завантаження

Завантажуються:

```text

```

Для кожної категорії.

# Консольний вивід

У стандартному режимі `INFO` консоль показує тільки основні операторські секції:

* параметри запуску;
* перевірку шляхів;
* перевірку вільного місця перед архівацією;
* архівацію та створення SHA512;
* перевірку SHA512;
* перевірку розміру архівів;
* SFTP-завантаження;
* retention архівів;
* очищення старих логів;
* підсумок виконання;
* шлях до лог-файлу.

Службові деталі, зокрема стан історії запусків і детальний health-блок резервних копій, винесені у `DEBUG` / лог-файл, щоб не перевантажувати звичайний консольний вивід.

## Summary Matrix / статус компонентів

Починаючи з `v2.6`, у фінальному підсумку скрипт виводить зведену матрицю стану для кожного компонента архівації.

Приклад:

```text
====================================================================================================
=== СТАТУС КОМПОНЕНТIВ ===

TYPE       ARCHIVE  SHA512  VERIFY   SIZE   RESTORE  SFTP   VALIDATION
---------- -------- ------- -------- ------ -------- ------ ----------
BLOG       OK       OK      OK       OK     OFF      OK     OK
VETOFFICE  OK       OK      OK       OK     OFF      OK     OK

====================================================================================================
```

Значення колонок:

| Колонка | Опис |
|---|---|
| `TYPE` | Компонент архівації. |
| `ARCHIVE` | Створення архіву. |
| `SHA512` | Створення SHA512-файлу. |
| `VERIFY` | Перевірка SHA512. |
| `SIZE` | Перевірка мінімального розміру архіву. |
| `RESTORE` | Test Restore, якщо він увімкнений. |
| `SFTP` | Завантаження на SFTP та перевірка розміру віддаленого файлу. |
| `VALIDATION` | Загальний статус локальної валідації архіву. |

Можливі статуси:

| Статус | Значення |
|---|---|
| `OK` | Етап виконано успішно. |
| `FAIL` | Етап завершився помилкою. |
| `OFF` | Етап вимкнений у конфігурації. |
| `DRY` | Етап пропущений через `DryRun`. |
| `PART` | Частковий успіх, наприклад частина файлів завантажена. |
| `N/A` | Немає даних для оцінки. |

# Історія змін

## v2.6 (2026-05-31)

### Додано

* Фінальну матрицю статусів компонентів `=== СТАТУС КОМПОНЕНТIВ ===`.
* Відображення статусів `ARCHIVE`, `SHA512`, `VERIFY`, `SIZE`, `RESTORE`, `SFTP`, `VALIDATION` для кожного компонента.
* Окремий статус `DRY` для режиму `DryRun`.
* Окремий статус `OFF` для вимкненого `Test Restore`.

### Покращено

* Фінальний операторський підсумок став зрозумілішим: тепер видно, який саме етап пройшов або впав.
* SFTP Verify показує розмір перевіреного файлу у байтах.
* Помилка форматування розміру у SFTP Verify виправлена.
## v2.5 (2026-05-30)

### Додано

* Перевірку розміру архіву після створення.
* Пороги контролю розміру архіву: мінімальний розмір у MB та мінімальна частка від розміру джерела.
* Опційний `Test Restore` архіву у тимчасову папку.
* Підсумковий статус `ArchiveValidationSuccess` для кожної категорії архівів.
* Заборону SFTP-завантаження та мережевого копіювання для архівів, які не пройшли повну валідацію.
* Оновлений комплект 7-Zip у папці `Tools`.
* Додано відображення версії 7-Zip у стартовому блоці параметрів.
* Параметри перевірки вільного місця перенесено в окремий блок перевірки диска.

* Службові повідомлення про JSON-звіт, історію запусків і health-блок резервних копій переведено в `DEBUG` / лог-файл.
* Покращено форматування SFTP-блоку, retention-блоку та підсумку виконання.

### Виправлено

* Зайві дубльовані роздільники у консольному виводі.
* Некоректне відображення `Test Restore`, коли він вимкнений.
* Нелогічне розміщення перевірки вільного місця наприкінці запуску.
* Перевантажений фінальний INFO-вивід.
