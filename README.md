# ARCHIV_VETOFFICE

Автоматизована система резервного копіювання для VetOffice з підтримкою архівації, перевірки цілісності, контролю вільного місця та завантаження резервних копій на SFTP-сервер.

---

# Версія

**ARCHIV_VETOFFICE v2.4**
Дата релізу: **2026-05-30**

---

# Основні можливості

## Архівація

* Створення архівів через 7-Zip.
* Підтримка паролів архівів.
* Підтримка SHA256 CRC.
* Багатопотокове стиснення.
* Перевірка архіву після створення (`7z t`).
* Контроль мінімального вільного місця перед архівацією.
* Автоматичне формування імен файлів.

Приклад:

```text
vetcontrol_dev_pnmgu_v2508_20260530_1657.mdz
```

---

## Контроль цілісності

Для кожного архіву автоматично створюється:

```text
archive.mdz
archive.mdz.sha512
```

Після створення виконується перевірка:

```text
Контрольна сума SHA512 збігається
```

# Завантаження на SFTP (нове у v2.4)

## Підтримувані сервери

* Hetzner Storage Box
* Стандартні SFTP сервери
* WinSCP Script Mode

---

## Особливості

### Автоматичне визначення StorageBox Host

Можна вказати:

```powershell
SftpHost = "u363066"
```

Скрипт автоматично перетворить на:

```text
u363066.your-storagebox.de
```

---

### Автоматичне створення каталогів

Перед завантаженням виконується:

```text
mkdir archiv
mkdir blog
```

Якщо каталог уже існує:

```text
File already exists
```

помилка ігнорується автоматично.

---

### Автоматичне завантаження

Завантажуються:

```text
archive.mdz
archive.mdz.sha512
```

Для кожної категорії.

---

### Детальна діагностика

У лог записуються:

```text
WinSCP stdout
WinSCP stderr
```

що значно спрощує пошук проблем.

---

## Статуси SFTP

### Успішно

```text

```

### Помилка підключення

```text

```

### Помилка завантаження

